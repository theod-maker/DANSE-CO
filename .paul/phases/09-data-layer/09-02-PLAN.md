## Pattern de conversion

Avant (Client Component) :
```tsx
'use client'
const disciplines = useDisciplines()
```

Après (Server Component) :
```tsx
import { sanityFetch } from '@/sanity/lib/fetch'
import { disciplinesQuery } from '@/sanity/lib/queries'
import { fallbackDisciplines } from '@/src/lib/fallbackContent'

const { data: disciplines } = await sanityFetch({
  query: disciplinesQuery,
  tags: ['sanity'],
}) ?? { data: fallbackDisciplines }
```

## Séparation Server / Client

Les pages deviennent Server Components. Les sous-composants avec :
- useState, useEffect, useRef, useInView → garder 'use client'
- Framer Motion (motion.*) → garder 'use client'
- Les cards, sections animées → garder 'use client'

Stratégie : extraire les parties animées en composants Client séparés si nécessaire.

    La page server passe les données comme props au composant client.

    Pattern :
    ```tsx
    // app/page.tsx — Server Component
    import { sanityFetch } from '@/sanity/lib/fetch'
    import { homepageQuery, newsQuery } from '@/sanity/lib/queries'
    import { fallbackHomepage, fallbackNews } from '@/src/lib/fallbackContent'
    import HomeContent from '@/src/components/home/HomeContent'

    export default async function Home() {
      const [{ data: homepage }, { data: news }] = await Promise.all([
        sanityFetch({ query: homepageQuery, tags: ['sanity'] }),
        sanityFetch({ query: newsQuery, tags: ['sanity'] }),
      ])

      return <HomeContent homepage={homepage ?? fallbackHomepage} news={news ?? fallbackNews} />
    }
    ```

    HomeContent.tsx contient le JSX animé existant de AsmeIndex.tsx avec 'use client'.
