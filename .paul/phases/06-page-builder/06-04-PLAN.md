## Goal
Migrer les 7 pages React vers le page builder + seeder les documents `page` dans Sanity avec le contenu actuel.

## Purpose
Les pages deviennent des shells qui : (1) essaient de récupérer leur contenu depuis Sanity via `usePage(slug)`, (2) si Sanity est vide ou non configuré, rendent le layout statique existant comme fallback.

## Output
- 7 pages React migrées avec fallback gracieux
- Script `scripts/seed-pages.mjs` qui popule Sanity avec les 7 pages
- Le client peut immédiatement modifier les blocs depuis le studio
</objective>

  <action>
    Pattern de migration pour chaque page :

    ```tsx
    const MyPage = () => {
      const pageData = usePage('/my-slug')  // null si Sanity vide

      return (
        <div className="min-h-screen overflow-x-hidden">
          <AppNavbar />
          {pageData ? (
            // Mode page builder — Sanity pilote le contenu
            <main className="pb-32">
              <BlockRenderer blocks={pageData.blocks} />
            </main>
          ) : (
            // Mode statique — fallback existant intact
            <main className="max-w-6xl mx-auto px-6 pt-40 pb-32 relative">
              {/* ... layout statique existant ... */}
            </main>
          )}
          <AppFooter />
        </div>
      )
    }
    ```

    **AsmeIndex.tsx** → slug: '/'
    **Disciplines.tsx** → slug: '/disciplines'
    **Instructors.tsx** → slug: '/instructors'
    **Planning.tsx** → slug: '/planning'
    **Pricing.tsx** → slug: '/pricing'
    **Locations.tsx** → slug: '/locations'
    **Contact.tsx** → slug: '/contact'

    Important : le layout statique existant est conservé INTACT dans la branche fallback. Rien n'est supprimé — on ajoute juste la branche `pageData ? <BlockRenderer> : <existant>`.
