<task type="checkpoint:human-action" gate="blocking">
  <what-built>Préparation déploiement — actions manuelles requises avant de continuer</what-built>
  <how-to-verify>
    Effectuer ces étapes dans l'ordre :

    **1. Créer le token Sanity "Viewer" :**
    - Aller sur sanity.io → projet z0uyxhwg → API → Tokens
    - Cliquer "Add API token"
    - Nom : "Vercel Preview"
    - Permissions : Viewer
    - Copier le token généré

    **2. Générer le secret de revalidation :**
    - Dans le terminal : `openssl rand -base64 32`
    - Copier le résultat

    **3. Mettre à jour .env.local avec les vraies valeurs :**
    ```
    SANITY_API_READ_TOKEN=<token copié en étape 1>
    SANITY_REVALIDATE_SECRET=<secret copié en étape 2>
    ```

    **4. Créer le projet Vercel :**
    - Aller sur vercel.com → New Project
    - Importer le repo GitHub de ce projet
    - Framework : Next.js (détecté automatiquement)
    - Ne pas encore déployer — passer à l'étape suivante d'abord
  </how-to-verify>
  <resume-signal>Taper "done" quand les 4 étapes sont complètes et que vous avez le projet Vercel créé (pas encore déployé)</resume-signal>
</task>

<task type="auto">
  <name>Task 4: Configurer le webhook Sanity</name>
  <files>— (configuration sanity.io)</files>
  <action>
    Sur sanity.io → projet z0uyxhwg → API → Webhooks → Create webhook :

    - Name : "Vercel ISR Revalidation"
    - URL : `https://VOTRE-URL.vercel.app/api/revalidate`
    - Trigger on : Create, Update, Delete
    - Filter : laisser vide (tous les types)
    - HTTP method : POST
    - HTTP Headers : aucun
    - Secret : coller la valeur de SANITY_REVALIDATE_SECRET
    - Enable webhook : ✓

    Tester le webhook avec le bouton "Send test notification" — doit retourner 200.
  </action>
  <verify>
    1. Modifier un contenu dans le Studio (ex: changer une description)
    2. Publier la modification
    3. Attendre 5-10 secondes

<task type="auto">
  <name>Task 5: Rédiger le guide client</name>
  <files>docs/guide-client.md</files>
  <action>
    Créer docs/guide-client.md avec le contenu suivant :

    ```markdown
    # Guide d'utilisation — Studio Danse&CO

    ## Accès au Studio
    URL : https://VOTRE-URL.vercel.app/studio
    Connexion : votre compte sanity.io (email + mot de passe)

    ## Modifier le contenu

    ### Voir l'aperçu avant publication
    1. Dans le Studio, cliquer sur l'icône "Presentation" dans la barre gauche
    2. Le site s'affiche en direct à droite
    3. Modifier un champ — le changement s'affiche immédiatement dans l'aperçu
    4. Cliquer "Publish" pour rendre le changement visible sur le site public

    ### Modifier une discipline
    1. Cliquer "💃 Disciplines" dans le menu gauche
    2. Sélectionner la discipline à modifier
    3. Modifier le titre, la description ou les bénéfices
    4. Cliquer "Publish"

    ### Modifier le planning
    1. Cliquer "📅 Planning des cours"
    2. Sélectionner le cours ou créer un nouveau avec "New"
    3. Remplir : nom, jour, heure, salle
    4. Cliquer "Publish"

    ### Modifier les tarifs
    1. Cliquer "💰 Tarifs"
    2. Modifier les lignes de tarif
    3. Cliquer "Publish"

    ### Ajouter une actualité
    1. Cliquer "📢 Actualités"
    2. Cliquer "New" en haut à droite
    3. Remplir titre, date, résumé, image
    4. Cocher "Publié"
    5. Cliquer "Publish"

    ### Modifier les informations générales (téléphone, email, réseaux sociaux)
    1. Cliquer "⚙️ Informations générales"
    2. Modifier les champs souhaités
    3. Cliquer "Publish"

    ## Délai de mise à jour
    Les modifications sont visibles sur le site dans les **10 secondes** après publication.

    ## Questions ?
    Contacter : [votre email de support]
    ```

    Remplacer VOTRE-URL et l'email de support.
  </action>
