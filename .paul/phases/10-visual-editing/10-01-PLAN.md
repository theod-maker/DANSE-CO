<objective>
## Goal
Activer le Presentation Tool dans le Studio (split-view éditeur + preview) et les overlays visual editing click-to-edit sur le site en mode draft.

## Purpose
Donner au client une expérience pro : il voit le site en temps réel pendant qu'il édite, et peut cliquer sur n'importe quel élément pour l'éditer directement.

## Output
- @sanity/presentation installé et configuré dans sanity.config.ts
- Stega activé dans sanity/lib/client.ts
- VisualEditing component dans app/layout.tsx (actif uniquement en draft mode)
- Checkpoint : vérification visuelle de l'expérience preview
</objective>

<context>
@.paul/PROJECT.md
@sanity.config.ts
@sanity/lib/client.ts
@app/layout.tsx

## Packages nécessaires
- @sanity/presentation (plugin Studio)
- @sanity/visual-editing (overlays site)

## Fonctionnement
1. Le client ouvre /studio → Presentation Tool affiche le site en iframe à droite
2. Pendant qu'il édite, les modifications apparaissent en temps réel (SanityLive)
3. En mode draft, des overlays verts apparaissent sur le site
4. Click sur un overlay → le Studio ouvre le bon champ
</context>

<task type="auto">
  <name>Task 4: Ajouter VisualEditing dans app/layout.tsx</name>
  <files>app/layout.tsx</files>
  <action>
    Créer d'abord un composant client src/components/layout/VisualEditingWrapper.tsx :

    ```tsx
    'use client'
    import { VisualEditing } from '@sanity/visual-editing/next'

    export default function VisualEditingWrapper() {
      return <VisualEditing />
    }
    ```

    Puis modifier app/layout.tsx pour l'inclure conditionnellement selon le draft mode :

    ```tsx
    import type { Metadata } from 'next'
    import { draftMode } from 'next/headers'
    import { SanityLive } from '@/sanity/lib/live'
    import VisualEditingWrapper from '@/src/components/layout/VisualEditingWrapper'
    import './globals.css'

    export const metadata: Metadata = {
      title: 'Danse&CO',
      description: 'École de danse sportive à Saint-Michel-Chef-Chef',
    }

    export default async function RootLayout({
      children,
    }: {
      children: React.ReactNode
    }) {
      const { isEnabled: isDraftMode } = await draftMode()

      return (
        <html lang="fr">
          <body>
            {children}
            <SanityLive />
            {isDraftMode && <VisualEditingWrapper />}
          </body>
        </html>
      )
    }
    ```

    VisualEditing n'est rendu que lorsque le draft mode est actif — pas d'impact sur les visiteurs normaux.
  </action>
  <verify>grep "VisualEditing\|isDraftMode" app/layout.tsx</verify>
  <done>AC-3 partiellement satisfait : VisualEditing dans le layout</done>
</task>
