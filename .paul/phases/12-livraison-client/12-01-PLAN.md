<objective>
## Goal
Livrer le site au client : nom de domaine, accès Studio, contenu réel, référencement Google.

## Purpose
Le client doit pouvoir utiliser son site de façon autonome — modifier le contenu, voir les changements en live, et être trouvé sur Google.

## Output
- Domaine custom connecté (ex: dansandco.fr)
- Client invité sur Sanity avec rôle Editor
- Google Search Console configuré
- Contenu réel saisi dans Sanity
- Client formé au Studio
</objective>

<tasks>

<task type="checkpoint:human-action" gate="blocking">
  <action>Choisir et acheter le nom de domaine avec le client</action>
  <instructions>
    Discuter avec le client du nom de domaine :
    - Suggestions : dansandco.fr, danse-and-co.fr, danseco44.fr
    - Acheter sur OVH (ovh.com) ou Namecheap (~10€/an pour .fr)
    - Le client crée le compte OVH à son nom (c'est son domaine)
  </instructions>
  <verification>Domaine acheté et accessible dans l'interface OVH</verification>
  <resume-signal>Taper "done" + le domaine choisi (ex: "done dansandco.fr")</resume-signal>
</task>

<task type="auto">
  <name>Task 2: Connecter le domaine sur Vercel</name>
  <files>sanity.config.ts</files>
  <action>
    Sur Vercel → projet DANSE-CO → Settings → Domains → Add Domain → saisir le domaine.
    Vercel affiche les DNS à configurer (2 enregistrements : A et CNAME).
    Sur OVH → Zone DNS → ajouter ces 2 enregistrements.
    Attendre 5-30 min pour la propagation.

    Puis mettre à jour sanity.config.ts :
    Remplacer 'https://danse-co.vercel.app' par 'https://NOM-DU-DOMAINE'
    dans la config presentationTool.previewUrl.origin.

    Commiter et pusher → Vercel redéploie avec le nouveau domaine.
  </action>
  <verify>https://NOM-DU-DOMAINE affiche le site avec un cadenas HTTPS</verify>
  <done>AC-1 satisfait : domaine custom opérationnel</done>
</task>

<task type="checkpoint:human-action" gate="blocking">
  <action>Inviter le client sur Sanity</action>
  <instructions>
    1. Aller sur https://www.sanity.io/manage/project/z0uyxhwg/members
    2. Cliquer "Invite member"
    3. Saisir l'email du client (professeur)
    4. Rôle : Editor (peut modifier le contenu, pas les schemas)
    5. Le client reçoit un email → crée son compte → accède au Studio

    Lui envoyer le guide : docs/guide-client.md
    URL du Studio : https://NOM-DU-DOMAINE/studio
  </instructions>
  <verification>Le client peut se connecter et voir le Studio</verification>
  <resume-signal>Taper "done" quand le client a accès</resume-signal>
</task>

<task type="checkpoint:human-action" gate="blocking">
  <action>Saisir le contenu réel dans Sanity</action>
  <instructions>
    Le client (ou toi avec lui) remplit le Studio avec le vrai contenu :

    Priorité haute :
    1. ⚙️ Informations générales → téléphone, email, adresse, Facebook
    2. 📅 Planning des cours → tous les cours avec horaires et salles
    3. 💰 Tarifs → prix réels saison 2026-2027
    4. 👩‍🏫 Professeurs → noms, bios, photos

    Priorité normale :
    5. 💃 Disciplines → descriptions personnalisées, vraies photos
    6. 🏛️ Salles → adresses vérifiées, photos
    7. 🏠 Page d'accueil → titre hero, description, photo de fond

    Chaque modification : cliquer "Publish" → le site se met à jour en quelques secondes.
  </instructions>
  <verification>Le site affiche le vrai contenu du club (pas les fallbacks)</verification>
  <resume-signal>Taper "done" quand le contenu principal est saisi</resume-signal>
</task>

<task type="checkpoint:human-action" gate="blocking">
  <action>Configurer Google Search Console</action>
  <instructions>
    1. Aller sur https://search.google.com/search-console
    2. "Add property" → saisir https://NOM-DU-DOMAINE
    3. Vérification : choisir "HTML tag" → copier la balise meta
    4. Me donner la valeur du content (ex: content="abc123...")
       → je l'ajoute dans le code en 2 minutes
    5. Cliquer "Verify" → Google confirme la propriété
    6. "Request indexing" sur l'URL principale

    Le site apparaîtra dans Google dans 1 à 4 semaines.
  </instructions>
  <verification>Google Search Console affiche "Property verified"</verification>
  <resume-signal>Taper "done" + la valeur meta (ex: "done abc123xyz")</resume-signal>
</task>
