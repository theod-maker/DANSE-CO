<objective>
## Goal
Ajouter une section Actualités sur la page d'accueil — entièrement configurable depuis Sanity. Chaque actu a un titre, une date, une image, un texte court et un lien optionnel.

## Purpose
Le client peut publier des nouvelles (stage exceptionnel, événement, info saison) sans toucher au code.

## Output
- Schema Sanity `news` (nouveau document répétable)
- Data layer : interface, fallback, query, hook
- Composant `NewsSection.tsx` affiché sur la page d'accueil
- Section masquée automatiquement si aucune actualité publiée
</objective>

    **sanity.config.ts** — ajouter dans la structure, après le divider final :
    ```

      .title('📢 Actualités')
      .child(S.documentTypeList('news').title('Actualités'))
    ```

    **src/lib/fallbackContent.ts** — ajouter :
    ```ts
    export interface NewsContent {
      _id: string
      title: string
      date: string
      imageUrl?: string
      excerpt: string
      link?: string
    }

    export const fallbackNews: NewsContent[] = [
      {
        _id: 'news-1',
        title: 'Reprise des cours — Saison 2025-2026',
        date: '2025-09-01',
        excerpt: 'La nouvelle saison démarre en septembre ! Inscriptions ouvertes lors des permanences.',
      },
      {
        _id: 'news-2',
        title: 'Stage Rock & Roll — Novembre',
        date: '2025-11-15',
        excerpt: 'Un week-end intensif pour progresser en Rock. Places limitées, inscrivez-vous rapidement.',
      },
    ]
    ```

    **src/lib/sanityQueries.ts** — ajouter :
    ```ts
    export const newsQuery = groq`*[_type == "news" && published == true] | order(date desc) {
      _id,
      title,
      date,
      "imageUrl": image.asset->url,
      excerpt,
      link,
    }`
    ```

    **src/hooks/useSanity.ts** — ajouter import + hook :
    ```ts
    export function useNews(): NewsContent[] {
      return useSanityData<NewsContent[]>(newsQuery, fallbackNews)
    }
    ```
